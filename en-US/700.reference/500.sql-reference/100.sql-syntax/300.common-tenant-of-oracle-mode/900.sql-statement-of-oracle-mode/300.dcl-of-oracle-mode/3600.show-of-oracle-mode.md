# SHOW

## Purpose

You can use this statement to query information about database objects.

## Syntax

```sql
SHOW {
   | TABLE STATUS
   | VARIABLES [like 'variable_name']
   | CHARSET | CHARACTER SET  
   | COLLATION
   | PARAMETERS
   | TABLEGROUPS
   | CREATE TABLEGROUP tablegroup_name
   | CREATE TABLE table_name
   | CREATE VIEW view_name
   | ERRORS
   | GRANTS
   | PRIVILEGES
   | RECYCLEBIN
   | [FULL] PROCESSLIST
   | TRACE [FORMAT='JSON']
};
```

## Parameters

| Parameter | Description |
|------------------------------------|----------------------------------------------------------------------------------------------------------|
| TABLE STATUS | Queries the details about all tables of the current user.  |
| VARIABLES \[like 'variable_name'\] | Queries the information about a variable. If `like 'variable_name'` is not specified, the information about all system variables is displayed. `variable_name` indicates the name of a variable.  |
| CHARSET \| CHARACTER SET | Queries supported character sets.  |
| COLLATION | Queries supported collations.  |
| PARAMETERS | Queries all system parameters.  |
| TABLEGROUPS | Queries table groups.  |
| CREATE TABLEGROUP tablegroup_name | Queries the `CREATE TABLEGROUP` statement. `tablegroup_name` indicates the name of the table group.  |
| CREATE TABLE table_name | Queries the `CREATE TABLE` statement. `table_name` indicates the name of the table.  |
| CREATE VIEW view_name | Queries the `CREATE VIEW` statement. `view_name` indicates the name of the view.  |
| ERRORS | Queries errors.  |
| GRANTS | Queries the privileges of the current user.  |
| PRIVILEGES | Queries the description of privileges.  |
| RECYCLEBIN | Queries the recycle bin.  |
| \[FULL\] PROCESSLIST | Queries the list of processes in the current tenant. The details are as follows:<ul><li><code>SHOW PROCESSLIST</code> displays the brief list of processes with the following information:  <ul><li><code>ID</code>: the ID of the process, that is, the client session ID of the current session. This ID is the unique identifier of the session in the client. </li><li><code>USER</code>: the username used for database connection. </li><li><code>HOST</code>: the IP address and port number of the client. If an OceanBase Data Proxy (ODP) is used to connect to the database, the IP address and port number of the ODP are displayed here. </li><li><code>DB</code>: the name of the connected database. </li><li><code>COMMAND</code>: the type of the command being executed, such as <code>Query</code> and <code>Sleep</code>. </li><li><code>TIME</code>: the execution time of the current command, in seconds. If the command is retried, the time is reset and recalculated. </li><li><code>STATE</code>: the status of the current session, such as <code>SLEEP</code> and <code>ACTIVE</code>. </li><li><code>INFO</code>: the content of the command being executed. A maximum of 100 characters can be displayed and the exceeding part is truncated. </li></ul>  </li><li><code>SHOW FULL PROCESSLIST</code> displays the complete list of processes with the following information:  <ul><li><code>ID</code>: the ID of the process, that is, the client session ID of the current session. This ID is the unique identifier of the session in the client. </li><li><code>USER</code>: the username used for database connection. </li><li><code>TENANT</code>: the connected tenant. </li><li><code>HOST</code>: the IP address and port number of the client. If an ODP is used to connect to the database, the IP address and port number of the ODP are displayed here. </li><li><code>DB</code>: the name of the connected database. </li><li><code>COMMAND</code>: the type of the command being executed, such as <code>Query</code> and <code>Sleep</code>. </li><li><code>TIME</code>: the execution time of the current command, in seconds. If the command is retried, the time is reset and recalculated. </li><li><code>STATE</code>: the status of the current session, such as <code>SLEEP</code> and <code>ACTIVE</code>. </li><li><code>INFO</code>: the content of the command being executed. </li><li><code>IP</code>: the IP address of the server. </li><li><code>PORT</code>: the SQL port number. </li></ul>  </li></ul>  <main id="notice" type='explain'><h4>Note</h4><p>You can execute the <code>SHOW PROCESSLIST</code> statement to query the quantity and IDs of sessions in the current database. For more information, see <a href="../../../../../1200.database-proxy/1500.view-tenant-sessions.md">View tenant sessions</a>. </p></main> |
| TRACE [FORMAT='JSON'] | Queries the execution status of SQL statements. You can choose to output the results in JSON format.  |

* View the information about the creation of the `tbl1` table.

   ```sql
   obclient> SHOW CREATE TABLE tbl1;

   | TABLE | CREATE TABLE                                                                                                                                                                          |

   +-------+---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
   1 row in set
   ```

* Check whether the recycle bin is enabled.

   ```sql
   obclient> SHOW VARIABLES LIKE 'recyclebin';
   +---------------+-------+
   | VARIABLE_NAME | VALUE |
   +---------------+-------+
   | recyclebin    | ON    |
   +---------------+-------+
   1 row in set
   ```

* View the objects in the recycle bin.

   ```sql

   +-----------------------------------+---------------+-------+------------------------------+
   2 rows in set
   ```
