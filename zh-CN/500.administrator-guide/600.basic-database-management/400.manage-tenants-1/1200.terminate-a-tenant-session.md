# 终止租户会话
#docslug#/oceanbase-database/oceanbase-database/V4.0.0/terminate-a-tenant-session
OceanBase 数据库支持通过 `KILL` 语句终止会话。

终止会话的 SQL 语句如下：

```sql
KILL [CONNECTION] 'session_id';
```

语句使用说明：

* `KILL` 语句每次只能终止一个会话。

* 如果您拥有 `PROCESS` 权限，则您可以查看所有会话。如果您拥有 `SUPER` 权限，您可以逐一终止所有会话和语句。否则，您只能查看和终止您自己的会话和语句。管理员可以逐一终止所有会话和语句。

  >**说明**
  >
  >* 查看当前拥有权限的操作请参见 [查看用户权限](500.manage-users-and-permissions/300.mysql-3/400.view-user-permissions-1.md)。如果您没有所需的权限，请联系管理员为您添加，为用户添加权限的相关操作请参见 [修改用户权限](500.manage-users-and-permissions/300.mysql-3/500.modify-user-permissions-2.md)。

终止指定的会话的示例如下：

```sql
obclient> KILL session_id;
```

或者

```sql
obclient> KILL CONNECTION session_id;
```

其中，`session_id` 即会话 ID，可通过 `SHOW PROCESSLIST` 语句或 `SHOW FULL PROCESSLIST` 语句获取。获取会话 ID 的详细操作及说明，请参见 [查看租户会话](../400.manage-tenants-1/1100.view-tenant-sessions.md)。

更多 `KILL` 语句的使用和说明，请参见 [KILL](../../../700.reference/200.sql-syntax/200.common-tenant-mysql-mode/600.sql-statement/3800.KILL.md)。
