# NEXT_DATE

`NEXT_DATE` 存储过程用于更改 Job 下次运行的时间。

  <main id="notice" >
    <h4>功能适用性</h4>
    <p>该内容仅适用于 OceanBase 数据库企业版。OceanBase 数据库社区版仅提供 MySQL 模式。</p>
  </main>

## 语法

```sql
DBMS_JOB.NEXT_DATE ( 
job             IN  BINARY_INTEGER,
next_date   IN  DATE);
```



## 参数解释



|    参数     |                                          解释                                          |
|-----------|--------------------------------------------------------------------------------------|
| job       | 预修改的 Job 对应的唯一 ID。 要查找此 ID，请查询 `USER_JOBS` 或 `DBA_JOBS` 视图的 `JOB` 列。 |
| next_date | 下次执行的时间。                                                                             |



## 使用说明

在提交之前，Job 将无法由后台的 Job 队列处理。
