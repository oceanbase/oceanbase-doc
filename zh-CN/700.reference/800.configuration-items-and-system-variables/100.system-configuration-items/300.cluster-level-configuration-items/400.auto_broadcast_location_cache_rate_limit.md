auto_broadcast_location_cache_rate_limit
=============================================================

`auto_broadcast_location_cache_rate_limit` 用于设置每个 OBServer 每秒可广播的位置信息变化的分区的最大数量。

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p> 从 V4.0.0 版本开始弃用。  </p>
</main>

|      **属性**      |    **描述**     |
|------------------|---------------|
| 参数类型             | 整型            |
| 默认值              | 1000          |
| 取值范围             | \[1, 100000\] |
| 是否重启 OBServer 节点生效 | 否             |

<main id="notice" type='notice'>
  <h4>注意</h4>
  <p> 该配置项需要在开启自动刷新位置缓存信息逻辑功能后才能生效。有关自动刷新位置缓存信息逻辑功能的相关配置项请参见 <a href="../300.cluster-level-configuration-items/6600.enable_auto_refresh_location_cache.md">enable_auto_refresh_location_cache</a>。</p>
</main>
