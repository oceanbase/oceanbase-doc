
# 系统变量总览

## Global 级别

### 升级相关

| 变量名 | 功能描述 |
|-------|---------|
| avoid_temporal_upgrade  | 用于管理时效性数据（如日期和时间）的升级行为。其默认值为 `OFF`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询，但无法设置。</p></main> |

### 存储引擎相关

| 变量名 | 功能描述 |
|-------|---------|
| default_tmp_storage_engine                                      | 用于指定默认的临时表存储引擎。其默认值为 `InnoDB`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询，但无法设置。</p></main> |
| innodb_tmpdir                                         | 用于指定 InnoDB 用于存储临时文件的目录。其默认值为 `""`，表示使用系统的默认临时目录。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询，但无法设置。</p></main> |
| ft_stopword_file | 用于指定停用词处理的文件路径。其默认值为 `built-in`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效。</p></main>                         |
| mecab_rc_file | 指定 MeCab 的配置文件路径。其默认值为 `""`，表示使用系统的默认路径。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效。</p></main>                         |
| innodb_temp_data_file_path | 指定 InnoDB 用于临时数据文件的存储路径。其默认值为 `ibtmp1:12M:autoextend`，表示 InnoDB 将使用名为 ibtmp1 的临时数据文件，初始大小为 12MB，并且该文件会根据需要自动扩展，以存储额外的临时数据。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效。</p></main>                         |
| innodb_data_file_path | 指定 InnoDB 数据文件的存储路径和文件名，其默认值为 `ibdata1:12M:autoextend`，表示 InnoDB 将使用名为 ibtmp1 的临时数据文件，初始大小为 12MB，并且该文件会根据需要自动扩展，以存储额外的临时数据。。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效。</p></main>                         |
| innodb_data_home_dir | 设置 InnoDB 数据文件的根目录，其默认值为 `""`，表示使用系统的默认路径。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效。</p></main>                         |

### 全文索引相关

| 变量名 | 功能描述 |
|-------|---------|
| innodb_ft_enable_diag_print                                        | 用于启用或禁用 InnoDB 全文索引的调试输出。其默认值为 `OFF`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询，但无法设置。</p></main> |
| innodb_ft_num_word_optimize                                        | 规定在进行 OPTIMIZE TABLE 操作时，需要操作的最小分词个数。其默认值为 `2000`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询，但无法设置。</p></main> |
| innodb_ft_result_cache_limit                                        | 用于设置 InnoDB 全文索引结果缓存的大小。其默认值为 `2000000000`，单位为字节。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询，但无法设置。</p></main> |
| innodb_ft_server_stopword_table                                     | 用于设置 InnoDB 全文索引的服务器停用词表。其默认值为 `NULL`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询，但无法设置。</p></main> |
| innodb_optimize_fulltext_only                                       | 用于控制 OPTIMIZE TABLE 语句在包含 FULLTEXT 索引的 InnoDB 表上的行为。其默认值为 `OFF`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询，但无法设置。</p></main> |
| max_tmp_tables                                        | 限制用户会话可以同时持有的临时表的最大数量。其默认值为 `32`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询，但无法设置。</p></main> |
| innodb_ft_cache_size | 设置 InnoDB 全文索引缓存的大小。其默认值为 `8000000`，单位为字节。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效。</p></main>                         |
| innodb_ft_sort_pll_degree | 用于全文索引排序的并行线程数，其默认值为 `2`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效。</p></main>                         |
| innodb_ft_total_cache_size | 设置 InnoDB 全文索引的总缓存大小，其默认值为 `640000000`，单位为字节。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效。</p></main>                         |
