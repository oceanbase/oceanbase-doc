
# 系统变量总览

## Global 级别

### InnoDB 存储引擎相关

| 变量名 | 功能描述 |
|-------|---------|
| innodb_large_prefix     | 用于设置是否对使用 DYNAMIC 或 COMPRESSED 行格式的 InnoDB 表，允许长度超过 767 字节（最多 3072 字节）的索引键前缀。其默认值为 `ON`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效。</p></main>                         |
| innodb_sort_buffer_size | 用于设置排序操作的缓冲区大小。其默认值为 `1048576`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效。</p></main>                         |

### MyISAM 存储引擎相关

| 变量名 | 功能描述 |
|-------|---------|
| delay_key_write          | 用于设置是否延迟对 MyISAM 表的索引键的写入。其默认值为 `ON`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效。</p></main>                         |
| key_buffer_size          | 用于设置 MyISAM 存储引擎的键缓冲区大小。其默认值为 `8388608`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效。</p></main>                         |
| key_cache_age_threshold  | 用于控制将缓冲区从密钥缓存的热子列表降级到温子列表的阈值。值越低，降级越快。其默认值为 `300`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效。</p></main>                         |
| key_cache_block_size     | 用于设置 MyISAM 键缓存中单个块的大小。其默认值为 `1024`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效。</p></main>                         |
| key_cache_division_limit | 用于设置 MyISAM 键缓存的划分限制。其默认值为 `100`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效。</p></main>                         |
| max_seeks_for_key        | 设置在 MyISAM 查询中进行最大寻址操作的次数。其默认值为 `18446744073709551615`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效。</p></main>                         |
